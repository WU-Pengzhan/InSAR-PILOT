# PLANS.md

## How We Work
1. Every non-trivial task starts with a short goal, constraints, and evidence section.
2. Decisions must cite local code evidence when they depend on ISCE behavior.
3. Milestones are closed only after code, validation, and docs are all updated.
4. Risks and open questions stay visible until resolved.

## Plan Template
- Goal
- Constraints
- Evidence
- Decision
- Implementation Steps
- Verification
- Risks / Open Questions
- Status

## Milestones
- M0: scaffolding and env validation
- M1: workflow generation and execution UI
- M2: result discovery and quicklooks
- M3: AOI options
- M4: adjacent-frame support if feasible

## Definition of Ready
- Inputs known
- Evidence gathered
- Scope bounded
- Verification approach identified

## Definition of Done
- Feature implemented
- Logs/errors understandable
- Tests or manual verification recorded
- Docs updated
